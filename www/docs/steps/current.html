OpenBSD: Описание процесса обновления системы до ветки -current

Мини-руководство "шаг за шагом"

Перед прочтением этого руководства, пожалуйста, ознакомьтесь cо списком важных изменений, которые были сделаны в OpenBSD ветки -current. Если вы хотите обновить систему до стабильной ветви, обратитесь к пошаговому руководству "OpenBSD: Описание процесса обновления системы до ветки -stable".

Указываем расположение общедоступного AnonCVS-сервера: 

# export CVSROOT=anoncvs@anoncvs.ca.openbsd.org:/cvs

Вот таким образом можно получить исходный код ветки -current: 

# cd /usr
# cvs -fqz3 checkout -P src

Чтобы произвести обновление исходного кода до -current: 

# cd /usr/src
# cvs -fqz3 update -PAd

Выполняем перекомпиляцию и установку нового ядра: 

# cd /usr/src/sys/arch/i386/conf
# config GENERIC
# cd ../compile/GENERIC
# make clean depend && make
# cp /bsd /bsd.old
# make install
# shutdown -r now

Переcобираем пользовательское пространство: 

# rm -rf /usr/obj/*
# cd /usr/src
# make obj build


Примечание 1. В некоторых случаях перед перекомпиляцией ядра может понадобиться удалить каталог /usr/src/sys/arch/i386/compile/GENERIC и в /usr/src выполнить цель includes: 

# rm -rf /usr/src/sys/arch/i386/compile/GENERIC
# cd /usr/src
# make includes

Примечание 2. В случае останова процесса пересборки системы вот так можно получить список уже выполненных команд: 

# make -n build

Для возобновления процесса необходимо повторно выполнить последнюю команду из этого списка, например: 

# make depend && make && make install

Примечание 3. Чтобы не root пользователь мог перекомпилировать ядро, убедитесь, что он входит в группу wsrc, и у него есть права на запись в директорию /usr/src: 

# chmod -R g+w /usr/src

Чтобы дать возможность пользователю builder производить перекомпиляцию системы, нужно соответствующим образом настроить sudo(8): 

# visudo
builder  ALL=(ALL)       NOPASSWD: ALL

И разрешить ему выполнение цели install во время работы ``make build'': 

# echo SUDO=sudo >> /etc/mk.conf

Примечание 4. Для того чтобы выполнить процесс перекомпиляции системы в фоновом режиме и перенаправить все данные, поступающие на стандартный выходной поток, в файл nohup.out, воспользуйтесь утилитой nohup(1): 

/usr/src# nohup make build &
[1] 10147
sending output to nohup.out

Если необходимо отслеживать бэкграундный процесс компиляции "в реальном времени", выполните следующую команду: 

# tail -f /usr/src/nohup.out
