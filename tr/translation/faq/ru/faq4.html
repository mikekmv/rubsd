4.1 - Обзор процедуры установки OpenBSD.

OpenBSD предоставляет несколько путей установки новой системы OpenBSD на различных архитектурах. Это позволяет установить систему в большинстве случаев без проблем. Существует несколько способов установки OpenBSD:

  • CD-ROM - Для этого Вам необходимо приобрести CD с операционной системой OpenBSD. Вы можете сделать это на стринице http://www.openbsd.org/orders.html Это пожалуй самый простой способ установки OpenBSD.
  • FTP - Этот способ требует знания Вашего IP, маски сети, адреса шлюза и т.п. Если у вас есть все необходимве данные, Вы може проинсталировать OpenBSD с одного из ftp-зеркал, расоложенных по всему миру, выбрав наиболее быстрое из них.
  • DHCP - DHCP - этот способ похож на FTP за исключением того, что IP, маска сети, адрес шлюза и т.п. получается с сервера DHCP.
  • Локальная файловая система - позволяет Вам проинсталлировать систему из файлов на существующей файловой системе. Инсталляционный диск поддерживает файловые системы DOS, EXT2FS и FFS.
  • NFS - позволяет установить систему с NFS-сервера. Обратитесь к руководству по mount_nfs(8) для более подробной информации.

Все эти способы доступны на одной инсталляционной дискете, которую можно скачать с любого ftp-зеркала ( список ftp-зеркал можно посмотреть на http://www.openbsd.org/ftp.html ). Каждая архитектура имеет свой собственный инсталляционный диск, поэтому перед инсталляцией убедитесть в правильности Вашего диска. Список поддерживаемых архитектур можно посмотреть в FAQ 1.

Ниже расположены ссылки полное пошаговое руководство по инсталляции для i386 и sparc архутектуры. Так же рекомендуется посмотреть INSTALL.i386 и INSTALL.sparc.

  • install-i386.html для i386.
  • install-sparc.html для sparc.
Люди, использующие другие архитектуры должны обратится к докуметнтам в INSTALL.arch, которые поставляются с дистрибутивом OpenBSD для соответствующих архитектур, а так же базовое руководство по инсталляции.

Замечания для Alpha-архитектур.

Alpha-система, на которую устанавливается OpenBSD, должна использовать SRM, а не ARC. Чтобы установить OpenBSD на alpha, Вы должны заагрузится с CD, используя CD2. Ниже приведена команда загрузки, которую Вы можете использовать:

    boot -fi 2.6/alpha/bsd.rd dkaX
Используйте опцию show device для поиска имени Вашего привода CDROM. Если Вы не можете загрузится с CD, Вы можете воспользоваться загрузочным диском CD2:2.6/alpha/floppy26.fs. Инструкции по созданию диска детально описаны в пошаговом руководстве по установке для архитектуры i386.

4.2 - Какие файлы нужны для инсталляции?

Существует несколько пакетов, содержащих файлы OpenBSD, какие из них нужны для установки и запуска системы? Ниже приведен обзор каждого пакета.

  • base26.tar.gz - Содержит базовые файлы системы. Этот пакет необходим.
  • etc26.tar.gz - Содержит все файлы из /etc Необходим.
  • comp26.tar.gz - Содержит компилятор и его утилиты и библиотеки. Рекомендуется для установки.
  • man26.tar.gz - Содержит электронное справочное руководство. Рекомендуется для установки.
  • misc26.tar.gz - Содержит дополнительную информацию, документацию по настройке.
  • game26.tar.gz - Содержит игры для OpenBSD.
  • xbase26.tar.gz - Содержит базовые файлы для X11
  • xfont26.tar.gz - Содержит шрифты для X11 и сервера шрифтов
  • xlink26.tar.gz - Has the X servers link kit
  • xserv26.tar.gz - Содержит X сервера для X11.
  • xshare26.tar.gz - Содержит дополнительные файлы для X
  • bsd - Ядро системы. Необходимо.

4.3 - Сколько места необходимо для установки OpenBSD?

Ниже предложены размеры подразделов файловой системы для полной установки системы. Они включают дополнительное пространство, которое позволит Вам запустить обычную домашнюю систему с доступом в Интернет.

  • Для многопользовательской системы Вам, вероятно, придется утроить эти значения.
  • Если Вы планируете проинсталлировать значительное количество дополнительного программного обеспечения, делайте /usr большим! Как минимум утройте эти значения!
  • Для системы, которая обслуживает множество почтовых ящиков или web-страниц (хранящихся соответственно в /var/mail и /var/www) Вам необходимо увеличить размер раздела /var.
  • Многопользовательская система может создавать большое количество лог-файлов (хранящихся в /var/log), поэтому лучше сделать раздел /var большим.
Так же помните, что /usr и /usr/X11R6 являются частями одной файловой системы, и разделение их на два раздела не дает большого преимущества. 
    SYSTEM          /       /usr    /var    /usr/X11R6
alpha           56M     540M    27M     161M
amiga           45M     399M    24M      36M
hp300           31M     234M    24M      47M
i386            35M     229M    24M      72M
mac68k          29M     232M    24M      36M
mvme68k         29M     232M    24M       -  (без X-сервера)
pmax            50M     355M    24M      60M
sparc           40M     259M    24M      49M

Когда Вы находитесь в редакторе диска, Вы можете создать систему которая будет иметь только раздел 'a' (основная файловая система) и 'b' (раздел подкачки). Файловая састема 'a' будет корневой системой, размер которой должен быть суммой 3 значений, представленных выше (/, /usr и /var) плюс некоторое пространство для /tmp. Раздел 'b' автоматически станет системным разделом подкачки -- рекомендуемый минимальный размер этого раздела 32М, если Ваш диск достаточно велик, сделайте этот раздел как минимум 64М. Если Ваш жесткий диск достаточно велик, увеличте этот раздел до 256М или даже до 512М.

Четыре важных причины использования отдельных файловых систем в отличие от единой файловой системы:

  • Безопастность: Вы можете пометить некоторые файловые системы как 'nosuid', 'nodev', 'noexec', 'readonly', и т.п.
  • Устойчивость: Пользователь или некорректная программа могут заполнить файловую систему мусором, если они будут иметь право записи на нее. Критические программы, запущенные на отдельных файловых системах, не прервут своей работы.
  • Скорость: Файловые системы, на которые часто происходит запись, могут дефрагментироваться (Файловая система ffs, которую использует OpenBSD, не склонна к сильной дефрагментации).
  • Целостность: Если одна из файловых систем выйдет из строя по какой-нибудь причине, то остальные файловые системы останутся неиспорченными.

4.4 - OpenBSD и другие ОС (i386, alpha)

OpenBSD & NT

Если Вы хотите работать с OpenBSD и NT, установленых на одном диске, Вы можете использовать NTloader - загрузчик который использует NT. Для этого Вам необходимо скопировать pbr OpenBSD на Вашем диске. После запуска installboot, Вы можете скопировать его например так: 

    	# dd if=/dev/rsd0c of=openbsd.pbr bs=512 count=1

Теперь загрузите NT и поместите openbsd.pbr на диск c:, затем добавьте следующую строчку в конец файла c:\boot.ini: 

    	c:\openbsd.pbr="OpenBSD"

После перезагрузки Вы сможете выбрать пункт OpenBSD из меню загрузки NT и загрузить OpenBSD. Более подробная информация о NTloader доступна в NTLDR Hacking Guide.

OpenBSD & Windows или DOS (i386)

Если Вы хотите работать с установленными OpenBSD, Window 3.1, Windows95 или DOS на Вашей машине, Вам необходимо использовать загрузчик, который сможет загружать все эти системы. Обратите внимание на osbs20b8.zip или The Ranish Partition Manager. Оба они имеют возможность загружать OpenBSD.

OpenBSD & Linux (i386)

Пожалуйста обратитесь к INSTALL.linux, который даст подробные инструкции об установке OpenBSD и Linux.

4.5 - Посылка dmesg@openbsd.org после инсталляции

Хотим напомнить, что для разработчиков OpenBSD важно знать какие устройства работают, а какие не очень.

Выдержка из /usr/src/etc/root/root.mail 

Если Вы полагаете, что OpenBSD работает лучше на Вашей машине, 
пожалуйста(после установки системы электронной почты) наберите 

         dmesg | mail dmesg@openbsd.org

чтобы мы могли видеть какую кофигурацию используют люди. Мы используем эту информацию
чтобы улучшить драйвера устройств в будущих версиях. (Было бы намного лучше если эта
информация была получена от ядра GENERIC (поставляемого с системой), а не от 
пользовательского ядра). Получаемая информация о драйверах устройств помогает нам 
улучшать работу драйверов.
См. также секцию 14.7

Убедитесь, что посылаете сообщение с аккоунта который также может получать почту, поскольку разработчики могут связаться с Вами, если у них будет что-то что они захотят проверить или изменить для улучшения работы системы. Не обязательно посылать сообщение с машины на которой была установлена OpenBSD, поэтому, если эта машина не может получать почту, просто наберите 

 dmesg | mail your-account@yourmail.dom
и затем перенаправьте это сообщение на адрес 
 dmesg@openbsd.org
где your-account@yourmail.dom это Ваш действующий почтовый аккоунт. (или перешлите результат dmesg с помощью ftp/scp/дискетой/...)

Внимание! - Пожалуйста, посылайте dmesg только от ядра GENERIC. Пользовательские ядра с удаленными из них устройствами мало чем могут помочь.
