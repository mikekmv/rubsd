Что такое анонимный CVS?

Анонимный CVS - это метод хранения вашей локальной копии OpenBSD дерева исходных текстов, относительно изменений сделаных в текущей версии OpenBSD. Также, возможно отслеживать заплаты к ошибкам допущенным в релизе.

Главное преимущество анонимного CVS относительно других возможностей обновлять исходной текст, в том, что он работает непосредственно с центральным репозиторием исходных текстов или его зеркалом. Это означает, что вы имеете полный набор CVS команд для управления обновлением и объединением ваших изменений с другими изменениями, вы можете вести историю изменений, находить различия, и выполнять прочие запросы.

Проект OpenBSD имеет три основных репозитория:

  • src - весь исходный текст для OpenBSD.
  • ports - адаптированное программное обеспечение (порты) для OpenBSD.
  • www - веб страницы OpenBSD. (Включая эту).

Сила использования анонимного CVS в том, что CVS является "приемлимой" системой контроля исходного текста, и она "уважает" ваши изменения, сделанные в локальном дереве исходных текстов и предпринит "наибольшие усилия" для обновления всего вашего дерева, нежели просто предоставить вам список неизвестных проблем, которые должны быть решены для продолжения.

Что такое CVS?

CVS - это система контроля версий для управления деревом исходных текстов OpenBSD. Она осуществляет центральный склад (репозиторий) для всего исходного текста, и его изменений. Разработчики имеют "право на запись" в центральном репозитории, что позволяет им менять его содержимое, пользователи анонимного CVS имеют "право на чтение" и могут получить исходные тексты разработчиков, не имея право изменить их на репозитории.

При этом позволяет разработчикам быть изолироваными друг от друга, каждый работает в своем собственном рабочем пространстве, а затем CVS объединяет законченные работы.

Когда изменения не могут быть полностью объединены, CVS обеспечивает "мягкий" переход в аварийный режим, и если вы сделали изменения в вашей локальной копии, CVS оставит немодифицированную копию вашей версии, продолжая модифицировать любые другие исходные модули, которые вы запросили.

Начинаем работу с анонимным CVS.

Самая последняя версия CVS доступна на Cyclic. Версии ранее чем 1.6 не рекомендуются, и могут быть неработоспособными. OpenBSD поставляется с CVS.

Те, у кого есть CD диск с OpenBSD, возможно, видели CVS/ директории на нем. Они нужны для продолжения обновления дерева. Используя это дерево, обновление с CVS репозитория будеть проходить значительно быстрее, нежели получение свежей копии полного древа. Есть два способа использовать этот CD:

  • Копировать CVS дерево с CD в /usr/src (предпологается CD примонтирован к /mnt): 
    	# cd /mnt; pax -rw CVS Makefile [a-z]* /usr/src
  • Использовать обединенное монтирование (см. mount_union(8)) CD ниже директории доступной для записи. (Это может использоваться только для компилирования дерева исходных текстов.) 
    	# mount -t union -o -b /mnt /usr/src

Для тех, у кого нет CD, могут использовать cvs(1) для копирования дерева исходных текстов с сетевого репозитория. Подробно об этом говорится в следующей главе.

После этого, /usr/src будет хорошим местом, где все cvs(1) команды должны будут работать.

Использование CVS для получения и обновления исходных текстов.

CVS был разработан, для простого восстановления и обновления ваших исходных текстов. Вы должны сначала решить, хотите вы следить за текущей веткой или веткой заплат (исправлений). Текущая ветка имеет все до минутных изменений, а ветка заплат содержит официальный выпуск OpenBSD + заплаты из errata

Как только вы решили за какой веткой следовать, вы должны выбрать CVS сервер. Список серверов приведен ниже. Имеется 3 способа доступа к этим серверам.

  • ssh - Secure Shell может использоваться, для доступа к анонимным CVS серверам. Мы рекомендуем этот способ, так как он использует шифрованный канал для передачи данных. Начиная с версии 2.6 OpenSSH включен в поставку OpenBSD.
  • rsh - Remote Shell может использоваться на некоторых CVS серверах для пользователей, которые не имеют доступ к ssh.
  • pserver - pserver полезен для пользователей, когда использование rsh или ssh неосуществимо (например, если сервер находится за файерволлом).

ОБРАТИТЕ ВНИМАНИЕ: Пользователи, желающие использовать ssh должны установить переменную окружения CVS_RSH.

  • Для Korn/Bourne оболочек. 
    	$ export CVS_RSH="/usr/bin/ssh"
  • Для CSH и его производных. 
    	$ setenv CVS_RSH /usr/bin/ssh

Как только вы определились, какой анонимный CVS сервер и какой способ доступа к нему вы будите использовать, вы готовы начать работу с cvs. Для тех у кого CD с OpenBSD могут использовать находяшееся на нем дерево исходных текстов, как описано выше. Если у вас нет CD, используйте способ приведенный ниже для получения исходных текстов. Этот способ поместит дерево исходных текстов в /usr/src. 

    # cd /usr; cvs checkout -P src

Вышесказанное скопирует текущее дерево исходных текстов OpenBSD. Чтобы скопировать дерево из ветки заплат, Вы должны указать тэг в вашей команде. Например: 

    # cd /usr; cvs checkout -P -rOPENBSD_2_7 src
Или OPENBSD_2_6 для 2.6, и т.д.

В настоящее время только тэг OPENBSD_2_7 содержит релиз исходных текстов + добавленные заплаты.

чтение и изменение для разработчиков:

Если вы зарегестрированы, у вас есть возможность работать с исходными текстами совместно с другими разработчиками. Ограничения получения исходных текстов криптографического ПО через CVS.

ВАЖНОЕ ПРИМЕЧАНИЕ: имеется несколько ограничений, касающихся криптографического программного обеспечения, которые вы должны знать:

  • Родина OpenBSD - Канада. Согласно исследованиям, проведенным гражданином Канады , и как значится в списке контроля над экспортом Канады, экспортировать криптографическое ПО в остальной мир РАЗРЕШЕНО.

  • Но, если вы находитесь вне США или Канады, вы не должны использовать криптографические секции OpenBSD с анонимных CVS серверов расположенных в США. Рассматриваемые файлы...

    • src/kerberosIV/*
    • src/lib/libdes/*
    • src/lib/libc/crypt/crypt.c
    • src/lib/libc/crypt/morecrypt.c
    • src/sys/netinet
    • src/usr.sbin/afs/src/rxkad/*
    • X11/xc/lib/Xdmcp/Wraphelp.c

    Криптографическое ПО из США может экспортироваться только в Канаду.

  • Проект OpenBSD нуждается в дополнительных анонимных CVS серверах, если вы хотите помочь, прочитайте дальше.

Пример работы с CVS.

Примеры использования анонимного CVS: 

    % setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
% cd /tmp
% cvs get src/sys/arch/sparc
    [ копирует файлы с репозитория на вашу машину ]
% cvs log src/sys/arch/sparc/sparc/locore.s
    [ показывает файл регистрации для заданного файла ]
% cvs diff -bc -r1.1 -r1.5 src/sys/arch/sparc/sparc/locore.s
    [ показывает изменения файла между версиями 1.1 и 1.5 ]

Чтобы подключится к серверу используя ''pserver'' (прямое tcp подключение вместо ssh или rsh) пользователь должен зарегистрироваться: 

        [ *ВНИМАНИЕ* Для этого вы должны использовать версию cvs 1.8 или выше    ]
% setenv CVSROOT :pserver:anoncvs@anoncvs.ca.openbsd.org:/cvs
% cvs login
(Logging in to anoncvs@anoncvs1.ca.openbsd.org)
CVS password: anoncvs
    [ При регистрации на сервере проверяется пароль, который затем ]
    [ сохраняется в файле ~/.cvspass (можно переназначить с        ]
    [ помощью переменной окружения CVS_PASSFILE) для дальнейшего   ]
    [ общения с сервером. Для примера, содержимое ~/.cvspass при   ]
    [ использовании пароля 'blah':                                 ]
    [ :pserver:anoncvs@anoncvs5.usa.openbsd.org:/cvs Au'yc         ]
    [ После регистрации вы можете заставить CVS соединяться        ]
    [ напрямую с сервером, используя при этом сохраненный пароль.  ]
% cvs get ksrc-i386 ksrc-common
    [ позволяет вам запросить файлы необходимое для компиляции ядра на i386  ]

Здесь приведен пример использования CVS для регулярного обновления исходных текстов:

  • Для начала, возьмем полное дерево исходных текстов:

    (Если вы следуете за текущей веткой): 

      # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
# cd /usr
# cvs -q get -PA src

    (Если вы следуете за веткой заплат для 2.7): 

      # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
# cd /usr
# cvs -q get -rOPENBSD_2_7 -P src
  • И потом, в любое время "обновим" это дерево:

    (Если вы следуете за текущей веткой): 

      # cd /usr/src
# cvs -q up -PAd

    (Если вы следуете за веткой заплат для 2.7): 

      # cd /usr/src
# cvs -q up -rOPENBSD_2_7 -Pd
    Каждый раз при обновлении, будет выполнятся синхронизация дерева в /usr/src. Ваши локальные изменения исходного текста не будут уничтожены, они сольются с другими изменениями. Если вы используете obj директории (не символические ссылки) вы можете добавить "-I obj" к команде запуска cvs, это задаст cvs выводить предупреждения обо всех встречаюшихся obj директориях которых нет в репозитории.

  • ПРИМЕЧАНИЕ: Если вы обновляете дерево исходных текстов, которое вы забрали с другого сервера или с CD, вы должны добавить -d $CVSROOT к команде запуска cvs. Вы так же должны установить переменную окружения CVS_IGNORE_REMOTE_ROOT. Если вы следуете за веткой заплат, убедитесь, что вы убрали флаг -A из команды cvs, в противном случае вы получите текущее дерево исходных текстов.

    (Если вы следуете за текущей веткой): 

      # cd /usr/src
# cvs -d $CVSROOT -q up -PAd

    (Если вы следуете за веткой заплат): 

      # cd /usr/src
# cvs -d $CVSROOT -q up -Pd

Обновление портов, аналогично src: 

  • # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
# cd /usr
# cvs -q get -PA ports
  • И потом, в любое время "обновим" это дерево: 
    # cd /usr
# cvs -q up -PAd ports

В этом примере, используется необязательный параметр -q, он необходим для того, чтобы уменьшить вывод команды cvs. Вы можете опустить этот параметр.

Создадим diff локально исправленного модуля (cd.c) для включения в отчет об ошибке: 

    # cd /usr
# cvs diff -u src/sys/scsi/cd.c > /tmp/patch

На странице руководства CVS(1) (поставляется с исходными текстами CVS) вы найдете гораздо больше информации о CVS и возможностях его использования.

Дерево исходных текстов X11
Сервера анонимного CVS также содержат исходные тексты X11. Вы можете получить дерево исходных текстов X11 со второго CD, как описано ниже:

  • скопировав дерево (предпологается 2й CD примонтирован к /mnt): 
            # cd /mnt; cp -Rp X11 /usr
  • используя объединенное монтирование ниже директории доступной для записи: 
            # mount -t union -o -b /mnt/X11 /usr/X11
После этого, /usr/X11 готов к использованию cvs. Для примера вы можете обновить исходные тексты до текущей версии (предпологается вы уже установили переменную окружения CVSROOT): 
            # cd /usr/X11
        # cvs -q update -PAd

Предупреждение: Будте внимательны при использовании cvs. Убедитесь что вы находитесь в root директории дерева или в таких директориях как /tmp. Некоторые команды, такие как "get" могут создать поддерево в текущей директории и последующее обновление рекурсивно flesh out this sub-tree.

Сервис анонимного CVS дает неопытным разработчикам возможность для изучения команд CVS и позволяет прочувствовать процесс разработки перед получением доступа "для разработчиков" -- как результата демонстрирования полезных навыков и высококачественных результатов. Люди создающие заплаты (исправления) могут создать их "diff"ы относительно дерева CVS, чтобы упростить интеграцию.

Что лучше использовать rsh(1) или ssh(1)?

Анонимный cvs: rsh против ssh
По умолчанию, CVS клиент использует протокол rsh для работы с CVS сервером. Большинство CVS серверов больше не поддерживают этот протокол по соображениям безопасности или например, если сервер находится за файерволлом или из-за несовершенства эмулятора протокола slirp. Все это может помешать вам использовать протокол rsh. Как альтернатива, использовать соединение с помощью "secure shell" это OpenSSH (свободный) или ssh (коммерческий и некоммерческий).

OpenBSD репозитории анонимного CVS поддержывают только протокол SSH1. SSH2 не поддержывается из-за "строгой" лицензии некомерческого использования.

Как только ssh установлен, измените переменную окружения CVS_RSH указывающаю на ssh (обычно /usr/bin/ssh). Если по каким-то причинам вы не можете соединится с 22 портом (стандартным для ssh) воспользуйтесь портом 2022.

При использовании ssh не включайте програмную компрессию (сжатие) передаваеммых данных т.к. CVS уже сжимает эти данные. Ниже приведен пример файла конфигурации $HOME/.ssh/config. И запомните, не все сервера анонимного cvs поддерживают ssh соединение на порт 2022. Обратите внимание, что большинство анонимных cvs серверов больше не принимают шифрование none, поскольку оно не используется в последних версиях ssh по соображениям безопасности. 

	Host anoncvs.ca.openbsd.org
	    Port 2022

При запуске, CVS немного болтлив; для того чтобы заставить его замолкнуть, вы можете сделать следующее: 

    setenv CVS_CLIENT_PORT -1

Если вы хотите сменить ваш CVS сервер на другой (скажем ваш отключили, или по какой-то другой причине) установите следующею переменную окружения: 

    setenv CVS_IGNORE_REMOTE_ROOT
При этом вы также должны использовать флаг -d, как сказано выше.

Зеркалируем cvs репозиторий используя sup(1).

Пользователи желающие создать зеркало OpenBSD CVS дерева могут сделать это с anoncvs.usa.openbsd.org. Обратите внимание, что это дерево cvs, а не извлеченное дерево исходных текстов. Это полезно, если вы хотите производить быстрые cvs операции (diff, annotate и т.д) или если у вас множество деревьев исходных текстов и вы только хотите переслать новые данные один раз (вы можете потом обновить дерево с вашего локального зеркала cvs).

Пример supfile'а: 

	cvs host=anoncvs.usa.openbsd.org hostbase=/ base=/home delete

Это заставит sup зеркалировать cvs дерево в /home/cvs и файлы данных sup в /home/sup. Полное OpenBSD cvs дерево около гигабайта в размере.

Установка зеркала анонимного cvs.

Если вы желаете создать новый анонимный cvs сервер, свяжитесь с sup@openbsd.org. Зеркала требуют около 1GB дискового пространства, и используют примерно 4MB свопа на одного пользователя (предпологается, что пользователь делает крупные операции; соответственно мелкие операции используют меньше ресурсов, но все равно, воздействие гораздо большее, чем при sup или ftp). Эти зеркала, должны иметь хорошую пропускную способность сети для области которую они собираются обслуживать. Подробнее об установке анонимного CVS сервера в этом документе.

Заключение:

После обновления исходных текстов и перед попыткой компиляции, прочитайте комментарии в начале файла /usr/src/Makefile. Желательно, чтобы вы откомпилировали и установили новое ядро перед исполнением make build. В некоторых случаях придется пересобрать и установить утилиту config для компиляции ядра. Если config GENERIC оканчивается неудачей, возможно это и есть тот самый случай.

Обратите внимание, что обновление релиза до текущего (находящегося в разработке) дерева, пересборкой исходных текстов, может быть довольно затруднительно из-за зависимостей, которые часто не очевидны. Поэтому, предпочтительней, если вы сначала установите самый последний снапшот, а уже потом попытаетесь обновить.
