Файловая структура:

Файловая структура каждого сервера примерно следующая:

  • .../OpenBSD/3.3/
    Здесь находятся наши самые последние релизы.
    • alpha/
    • amiga/
    • hp300/
    • hppa/
    • i386/
    • mac68k/
    • macppc/
    • mvme68k/
    • sparc/
    • sparc64/
    • vax/
    • src.tar.gz
       Исходные тексты пользовательских программ, которые вы обычно помещаете в /usr/src. (В их состав не входят исходные тексты ядра)
    • sys.tar.gz
       Исходные тексты ядра, их вы обычно помещаете в /usr/src/sys.
    • ports.tar.gz
       Исходные тексты портов, вы обычно размещаете их в /usr/ports.
    • X11.tar.gz
       Исходные тексты XFree86 X11.
    • XF4.tar.gz
       Исходные тексты XFree86 (X11).

  • .../OpenBSD/snapshots/
    Для наших основных архитектур мы примерно раз в месяц создаём минирелизы неизвестной стабильности и качества. Эти релизы мы размещаем здесь.

  • .../OpenBSD/src/
    Здесь находится ежедневная выборка из cvs исходных текстов для тех, кто хочет иметь доступ к отдельным файлам. Обычно мы рекомендуем, использовать CTM или AnonCVS. Так как этот каталог не зеркалируется.

  • .../OpenBSD/doc/
    OpenBSD FAQ в PDF и текстовом форматах, а так же другая относящаяся к OpenBSD документация.

  • .../OpenBSD/tools/
    Различные утилиты.
  • .../OpenBSD/3.2/
    Структура как и у 3.3, описана выше.. для ностальгирующих.
  • .../OpenBSD/3.1/
    Структура как и у 3.3, описана выше.. для ностальгирующих.
  • .../OpenBSD/3.0/
    Структура как и у 3.3, описана выше.. для ностальгирующих.
  • .../OpenBSD/2.9/
    Структура как и у 3.3, описана выше.. для ностальгирующих.
  • .../OpenBSD-ISO/
    ISO образы дополнительных частей, не поставляемых с набором дисков. В настоящий момент содержит только 4-ый CD OpenBSD 3.1.
    Примечание: ftp зеркала обычно не зеркалируют эту директорию.

Тем, кто создаёт новые зеркала FTP/ANONCVS, может быть интересен примерный размер занимаемого дискового пространства на ftp://ftp.openbsd.org:/pub/OpenBSD/.
Зеркала перечисленные на этой странице, должны предоставить доступ по крайней мере на два последних релиза в бинарном виде (на текущий момент, это 3.2 и 3.3) для всех архитектур, а так же директории OpenSSH/, patches/ и tools/.
В зависимости от свободного места, зеркала могут также предоставлять snapshots, старые релизы, дерево исходных текстов и т.д.
Используемое место на диске, на главном ftp сервере: 

                       сейчас  может быть
pub/OpenBSD/2.9        3.0GB     3.0GB   (релиз OpenBSD 2.9)
pub/OpenBSD/3.0        4.5GB     4.5GB   (релиз OpenBSD 3.0)
pub/OpenBSD/3.1        6.9GB     6.9GB   (релиз OpenBSD 3.1)
pub/OpenBSD/3.2        11.8GB    11.8GB  (релиз OpenBSD 3.2)
pub/OpenBSD/3.3        14.0GB    14.0GB  (релиз OpenBSD 3.3)
pub/OpenBSD/snapshots  8.5GB     больше! (текущий снапшот, обычно >=
                                         размеру последнего дистрибутива)
pub/OpenBSD/src        .6GB      .7GB    (дерево src, доступное по FTP,
					 создается при помощи anoncvs каждую
					 ночь. Вы не должны использовать
					 зеркала для его получения,
					 лучше скопировать его с ближайшего
					 anoncvs сервера)
Для полного зеркала необходимо около 50GB, а для частичного (последние два релиза) только 25GB.

Кроме того, главный ftp сервер предоставляет следующие директории: 

pub/OpenBSD/distfiles  4.4GB    больше!	(distfiles используются подсистемой
					портов. Вы вероятно не должны
					зеркалировать их, если только для
					своих нужд. Make-файлы из портов по
					умолчанию будут искать их на
					ftp.openbsd.org и ftp.usa.openbsd.org)
pub/OpenBSD-ISO        0.7GB   bigger!	(доступный только по ftp, cd-rom ISO
					image для m68k платформ)

Если вы создали новое ftp, http, AFS или rsync зеркало, пожалуйста, напишите в www@openbsd.org.
